# Fundamentos do Gerenciamento de Backlog para o Backlog do Produto e do Sprint

Esta seção contém links que direcionam você para as melhores práticas de gerenciamento de Backlogs do Produto e do Sprint. Após ler as melhores práticas, você deve ter um entendimento básico de como gerenciar ambos os backlogs do produto e do sprint, como criar critérios de aceitação para histórias de usuários, criar uma definição de "feito" e definição de "pronto" para histórias de usuários e os fundamentos sobre a estimativa de histórias de usuários.

- [Backlog do Sprint](https://scrumguides.org/scrum-guide.html#sprint-backlog)
- [Critérios de Aceitação](https://learn.microsoft.com/en-us/azure/devops/boards/backlogs/best-practices-product-backlog?view=azure-devops#acceptance-criteria)
- [Definição de Feito](https://scrumguides.org/scrum-guide.html#increment)
- [Definição de Pronto](https://www.scrum.org/resources/blog/walking-through-definition-ready)
- [Fundamentos da Estimativa em Ágil](https://www.scrum.org/resources/blog/what-scrum-says-about-estimates)
